Automatically unpublish or delete posts after a specified expiration date.

== Description ==

Auto-Expire Posts allows content managers to set an expiration date on posts, so they automatically unpublish when outdated.

**Features:**

* Set expiration date and time for posts
* Choose to set posts to draft or delete them on expiry
* Simple and intuitive interface in the post editor
* No configuration needed - works out of the box

Perfect for:
* Time-limited promotions
* Event announcements
* Seasonal content
* Limited-time offers

== Installation ==

1. Upload the `auto-expire-posts` folder to the `/wp-content/plugins/` directory
2. Activate the plugin through the 'Plugins' menu in WordPress
3. Edit any post to set an expiration date

== Frequently Asked Questions ==

= How do I set a post to expire? =

When editing a post, look for the "Post Expiration" meta box in the sidebar. Set your desired expiration date and time, and choose whether to set the post to draft or delete it when it expires.

= Will this work with custom post types? =

Currently, the plugin only works with standard WordPress posts. Support for custom post types may be added in future versions.

= What happens if I don't set an expiration date? =

If no expiration date is set, the post will remain published indefinitely (standard WordPress behavior).
